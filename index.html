How Sudoku Solving Works — DSA Under the Hood

🔢 2D Array (Matrix Representation)

The Sudoku board is stored as an N×N 2D Array (Matrix). Rows, columns, and subgrid boxes are traversed and validated using simple mathematical row/column coordinate offsets.

• O(1) Access: Constant-time access to retrieve or update symbols at any (row, col).
• O(N) Traversal: Checks rows, columns, and subgrids in linear time to confirm value safety.
💻 View isValid() Code 
function isValid(board, r, c, val, cfg) {
  const n = cfg.size, bR = cfg.bR, bC = cfg.bC;
  for(let i=0; i<n; i++){
    if(i!==c && board[r][i]===val) return false;
    if(i!==r && board[i][c]===val) return false;
  }
  const sr = Math.floor(r/bR)*bR, sc = Math.floor(c/bC)*bC;
  for(let i=sr; i<sr+bR; i++)
    for(let j=sc; j<sc+bC; j++)
      if((i!==r||j!==c) && board[i][j]===val) return false;
  return true;
}

• State Stack: The browser call stack stores local cell variables and guesses for each recursive step.
• Base Case: Solving succeeds and unwinds when no empty cells remain.

🔙 Backtracking (Depth-First Search)

Backtracking explores possible numbers. If a path violates constraints, the algorithm "backtracks" (rewinds) by resetting the cell value to 0, pruning the path and trying the next choice.

• Pruning: Eliminates dead-ends early, avoiding brute-force exploration of all $9^{81}$ grid paths.
• Rollback: Clears grid values automatically when a recursive call returns failure.
💻 View Backtrack Loop Code 
for(const s of syms) {
  if(isValid(board,r,c,s,cfg)) {
    board[r][c] = s; // Make choice
    if(solve(board,cfg)) return true;
    board[r][c] = 0; // Backtrack (undo choice)
  }
}

⚡ MRV Heuristic (Pruning Optimization)

The Minimum Remaining Values (MRV) heuristic identifies the cell with the fewest valid candidate numbers (the most constrained cell) and solves it first.

• Constraint Scaling: Discovers invalid paths immediately, preventing deep, useless backtracking tree layers.
• Huge Speedup: Accelerates solver performance by over 1000×, making larger grid solving feasible.
💻 View findBestEmpty() Code 
function findBestEmpty(board, cfg) {
  let best=null, min=cfg.size+1;
  for(let r=0; r<cfg.size; r++) {
    for(let c=0; c<cfg.size; c++) {
      if(board[r][c]===0) {
        let cnt=0;
        for(const s of cfg.syms) 
          if(isValid(board,r,c,s,cfg)) cnt++;
        if(cnt<min){ min=cnt; best=[r,c]; }
        if(cnt===0) return best; // Fail early
      }
    }
  }
  return best;
}

📥 Stack (LIFO Undo History)

The application tracks moves using a Stack (LIFO - Last In, First Out). Every cell entry pushes a state item onto the stack, enabling sequential move rollbacks.

• O(1) Push/Pop: Reversing the board state uses constant-time stack updates.
• State Recovery: Easily restores previous cell values on click of the Undo button.
💻 View Stack Undo Code 
function handleUndo() {
  if (S.history.length === 0) return;
  const move = S.history.pop(); // Pop from stack
  S.board[move.r][move.c] = move.old;
  drawBoard();
  updateStats();
}

🎲 Fisher-Yates Shuffle

Unbiased random puzzle generation is powered by the Fisher-Yates Shuffle. It randomly permutes coordinates and candidates to generate puzzles.

• O(N) Complexity: Shuffles the array elements in place in linear time.
• Uniform Distribution: Ensures all board permutations have equal probability.
💻 View Shuffle Algorithm Code 
function shuffle(a) {
  for (let i = a.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [a[i], a[j]] = [a[j], a[i]]; // Swap elements
  }
}

🕸 Graph Vertex Coloring

Sudoku can be modeled as the Graph Vertex Coloring Problem. Cells represent vertices, and edges represent constraint relationships in rows, columns, or subgrid cliques.

• Constraint Clique: A vertex coloring problem where neighboring vertices cannot share the same color.
• NP-Complete Logic: Links game solving to foundational computational complexity theory.
